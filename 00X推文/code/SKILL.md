# X 推文深度分析 Skill

你的任务是分析用户提供的 X(Twitter) 推文原始数据，生成一份高质量的深度分析报告。

## 核心原则

1. **每个人的观点都必须被总结，不能遗漏** — 这是最重要的规则。哪怕只有一条推文，也要在报告中体现。宁可报告长一点，也不能丢掉任何人的声音。
2. **区分事实与观点** — 引用原文时标注是数据/事实还是个人判断
3. **保留原文精华** — 对关键观点直接引用原推文（英文保留英文，中文保留中文），不要过度意译
4. **逻辑链要完整** — 不只是罗列观点，要把"因为→所以→意味着"的推理链条理清楚
5. **多空必须对称** — 不能只讲看多不讲看空，反之亦然

## 数据格式

输入文件是 `.txt` 格式，每条推文的结构如下：

```
[推文 N]
发言人: 用户名 @handle · 时间描述
时间: ISO时间戳
推文: 推文内容

[📌 引用旧内容]：
被引用的推文内容（可选）

========================================
```

## 分析步骤

### 第一步：数据预处理

1. 逐条读取所有推文
2. 按发言人归类，统计每人发了多少条
3. 识别哪些推文是同一话题的讨论（通过引用关系、关键词、@提及）
4. 过滤纯转发/无实质内容的推文（如只有"."或单个emoji的）

### 第二步：话题聚类

将所有推文按话题聚类，一条推文可能属于多个话题。聚类时：
- 先找高频关键词和 $股票代号
- 再找引用/回复链条
- 最后用语义相似性归类剩余推文

### 第三步：按大佬名单分类

读取大佬名单文件来辅助分类：
- AI 大佬名单：`01周报日报/00X推文/code/AI 大佬.md`
- 投资大佬名单：`01周报日报/00X推文/code/投资大佬.md`

如果推文的发言人在名单中，标注其身份。不在名单中的发言人也要正常分析，不要忽略。

### 第四步：生成报告

按照下面的报告模板逐节撰写。

## 质量检查清单

报告写完后，按这个清单自查：

- [ ] 每个发言人的每条有实质内容的推文都被覆盖了吗？
- [ ] 多空观点是否对称呈现？有没有只讲一面？
- [ ] 逻辑链是否完整？"因为→所以→意味着"都写清楚了吗？
- [ ] 原文引用是否准确？有没有过度意译？
- [ ] 情绪指标是否基于数据而非主观感觉？
- [ ] 投资话题是否按热度排序？
