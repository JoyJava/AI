# 每日早报 · 编排层

## 何时使用本 skill

- 用户说「生成今日早报 / 来份早报 / 帮我看下盘前 / 晨报 / morning brief」等
- 用户说「整合一下今天的新闻、宏观、拥挤交易、大佬仓位」
- 定时任务（建议每日 7:30）触发，自动生成当日 HTML

如果用户**只**问其中某一个模块（例如只问"大佬仓位"），不要触发本 skill，让对应的子 skill 自己处理。本 skill 只在用户要"早报 / 综合 / 整合 / 全部 / 一份报告"这类整合需求时触发。

---

## 输出规格

**唯一产物**：单个 HTML 文件
**路径**：`/{YYYY-MM-DD}早报.html`
（`{YYYY-MM-DD}` 用本地时间，先 `date +%F` 取一次，全文统一）

HTML 必须满足：

1. **顶部时间戳条**：生成时间（含时区）、数据基准日期、4 个模块各自的"数据截至时间"
2. **顶部锚点导航**：`新闻情报 / 宏观诊断 / 拥挤交易 / 大佬仓位`，点击跳转
3. **每模块独立 section**，标题 + 数据时间戳 + 内容 + "原始来源链接"小字
4. **数据缺失红条横幅**：任何子模块抓取失败，顶部显示红色横幅标明「X 模块数据缺失，原因：…」
5. **风格**：参考同目录下 `分析结论/拥挤交易追踪.html` 的配色和排版（深色背景、卡片式 section、关键数字高亮）。如果该文件存在，先 `Read` 一下抄它的 CSS 框架
6. **footer**：列出本报告调用的 4 个 skill 名 + 数据源清单（FMS/COT/SEC EDGAR/Polymarket 等）

## 编排流程

### 第 0 步 · 锁定时间

```bash
date +"%F %H:%M %Z"
```

把这个时间戳作为「报告生成时间」全文复用，避免子任务之间漂移。

### 第 1 步 · 检查既有产物，决定 full vs incremental

```
target = [工作空间]/{今日}早报.html
```

- 如果 target **已存在**（同一天再次触发）→ Read 一下，看哪些模块时间戳超过 4 小时，**只**重跑过期模块（节省成本）
- 如果 target **不存在** → 4 个模块全跑

### 第 2 步 · 4 个并行子任务

**关键：必须用一条消息发出 4 个 Agent tool calls，让它们真正并行。** 主线程不要自己跑这些 skill，会污染上下文且无法并行。

每个子 agent 用 `general-purpose` 类型，prompt 自包含（子 agent 看不到本对话）。模板：

```
Agent #1 — news-briefing
  description: "今日新闻情报快报"
  prompt: |
    今天是 {YYYY-MM-DD}（北京时间）。请调用 news-briefing skill，
    生成过去 24 小时的全球新闻情报快报，输出 HTML 片段（<section> 级别，
    不要 <html>/<head>），覆盖：国际要闻、宏观财经、加密货币、美股动态、
    大佬观点、AI 行业动态、Polymarket。每条新闻末尾保留原始来源链接。
    最后单独输出一行 `DATA_TIMESTAMP: <数据截至 UTC 时间>`。

Agent #2 — 美股宏观分析框架
  description: "美股宏观诊断（K 型/传导链）"
  prompt: |
    今天是 {YYYY-MM-DD}。请用「美股宏观分析框架」skill 的模式 A（诊断分析师），
    生成当前传导链所处阶段的简短诊断（不要完整长报告，只要早报用的精华版：
    当前阶段编号、3 个关键领先指标读数、是否触发减仓信号）。
    输出 HTML <section> 片段。末尾给 `DATA_TIMESTAMP`。

Agent #3 — 美股拥挤交易追踪
  description: "拥挤交易四层数据快照"
  prompt: |
    今天是 {YYYY-MM-DD}。请用「美股拥挤交易追踪」skill，
    给早报用的精简版：① 当月 BofA FMS 最拥挤的 3 个交易；
    ② 最新 CFTC COT 杠杆极值；③ 过去 7 天 OBV 资金流向异常的 ETF。
    不要重新生成 拥挤交易追踪.html，只要 HTML <section> 片段供早报嵌入。
    末尾给 `DATA_TIMESTAMP`。

Agent #4 — 美股逃顶大佬仓位
  description: "大佬仓位变化追踪"
  prompt: |
    今天是 {YYYY-MM-DD}。请用「美股逃顶大佬仓位」skill 的数据验证协议，
    给早报用的精简版：过去一周内有公开仓位/言论变化的大佬清单
    （每人 1 行：变化方向、关键持仓、出处链接）。强调"行动 vs 语言"差异。
    输出 HTML <section> 片段。末尾给 `DATA_TIMESTAMP`。
```

### 第 3 步 · 主线程合成 HTML

收到 4 段 `<section>` 后：

1. 解析每段末尾的 `DATA_TIMESTAMP`，写进顶部时间戳条
2. 任何子任务返回错误或 timestamp 缺失 → 在顶部红条横幅列出，对应 section 用占位符「⚠️ 数据缺失」
3. 套用统一 CSS（沿用 `拥挤交易追踪.html` 风格）
4. 用 Write 写到 `01周报日报/{YYYY-MM-DD}早报.html`
5. 在对话里给用户**一句话**总结：「早报已生成 → file path。今日关键信号：[1-2 句最重要的发现]」

## 失败模式与降级

| 情况 | 处理 |
|---|---|
| 1 个子 agent 失败 | 红条横幅 + 该 section 占位符，其余正常输出 |
| 2+ 子 agent 失败 | 顶部红条警告，HTML 仍生成，提醒用户人工核查 |
| 全部失败 | 生成「今日数据抓取异常」占位 HTML，附手动核查清单（4 个 skill 各自的一手来源 URL） |
| 用户在非交易日触发（周末） | 正常生成，但顶部标注「美股休市，价格/COT 数据沿用上一交易日」 |

---

## 不要做

- ❌ **不要**主线程自己依次跑 4 个 skill（串行 + 上下文污染）
- ❌ **不要**生成 4 份独立文件（用户要的是单一 HTML）
- ❌ **不要**在没有一手来源的情况下编造数字 — 沿用各子 skill 的「数据验证协议」
- ❌ **不要**每次都重写已有的 `拥挤交易追踪.html` / `大佬仓位追踪.html` 单独报告 — 早报只嵌入精简版片段，不动那两个独立产物
- ❌ **不要**写"价格目标 / 涨跌预测" — 早报是事实层和信号层，不是荐股

## 与定时任务的衔接

如果用户后续问「能不能每天早上自动生成」：建议用 launchd（macOS）或 cron 在每天 7:30 跑一条 `claude -p "/skill 每日早报"`（具体命令以 Claude Code CLI 实际语法为准）。本 skill 本身只负责被触发后的产出，不内置调度逻辑。
