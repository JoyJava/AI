# 美股宏观分析框架

## 何时使用本 skill

- 用户询问当前美股宏观状况、是否接近衰退、要不要调仓
- 用户提到：传导链 / K 型分化 / 幸存者偏差 / 发薪日效应 / 信用周期 / 阶段（N）
- 用户要求生成综合诊断报告、月度追踪、财报日历摘要
- 周五 09:03 的定时任务触发（见"周五工作流"）

## 两种模式

调用时先判断用户意图，选对应模式：

### 模式 A：诊断分析师（默认）
用户问"现在什么情况" / 周五定时触发 → 产出一份**当前诊断报告**。

### 模式 B：框架讲解员
用户问"这框架怎么用 / 阶段 X 是什么意思 / 为什么看这个指标" → 解释框架内部逻辑，**不生成报告**。

---

## 核心原则（所有分析必须遵守）

1. **看领先指标，不看滞后指标**  
   JOLTS 辞职率、招聘率、次级信贷准入率、银行贷款损失准备金 → 这些**领先失业率和 GDP 6-12 个月**。失业率亮红灯时市场往往已经跌完。

2. **看分层数据，不看平均数**  
   个人储蓄率是平均数，被高收入拉高；必须看旧金山联储**分层超额储蓄**。工资增速看底层 20% 分位数，不看均值。

3. **警惕幸存者偏差（核心反直觉点）**  
   信用卡逾期率"变好"≠ 信贷质量好，可能是银行把次级借款人赶出系统（分母缩小）。必须交叉验证：
   - 绝对逾期金额（不看比率）
   - 次级专营商（SYF、Bread Financial）独立数据
   - 次级信贷准入率（FICO<660 新发占比）
   - 影子信贷（BNPL 食品占比、EWA 规模）

4. **分子/分母都要拆**  
   每个比率指标都要问："分子为什么变？分母为什么变？" 比如信用卡逾期率下降可能是分子（逾期金额）真降，也可能是分母（总余额）被银行收紧膨胀前值、或次级余额被清出。详见 `references/指标详解.md`。

5. **对比"周期演变"而非"单点快照"**  
   参照列使用 **1995-2005**（dotcom 全周期，每年一值）和 **2003-2008**（次贷积累期，每年一值）。对比**轨迹**比对比**峰值**更能揭示当前位置。

## 11 阶段传导链（一句话版）

| # | 阶段 | 核心问题 |
|---|------|---------|
| 1 | 收入端压力 | 底层是否还有足够现金流入？ |
| 2 | 储蓄耗尽 | 没有流入后，家底还能撑多久？ |
| 3 | 举债求生 | 开始借钱维持生活了吗？ |
| 4 | 消费降级 | 砍哪些支出？（4a 奢侈 → 4h 食品援助）|
| 5 | 系统内信用压力 | 信用卡/汽车贷逾期率在升吗？|
| 6 | 被踢出系统 | 银行开始拒绝次级借款人了吗？|
| 7 | 硬资产损失 | 汽车被收回、房屋止赎了吗？|
| 8 | 正式退出 | 破产申请在加速吗？|
| 9 | 银行暴露 | 损失传到银行资产负债表了吗？|
| 10 | 实体经济 | 企业裁员、盈利下调了吗？|
| 11 | 市场重定价 | VIX/信用利差/估值修正开始了吗？|

**详细的每阶段结论/思路/行动** → 见 `references/传导链11阶段.md`

## 标准工作流

### 工作流 A：生成一份综合诊断报告（默认）

1. **读取框架文件**
   - `references/传导链11阶段.md` — 每阶段的结构化结论
   - `references/指标详解.md` — 分子/分母拆解
   - `references/决策规则.md` — 仓位对照表 + 触发器
   - `references/财报日历.md` — 本周哪些关键公司发财报
   - `references/历史验证.md` — 对比历史周期

2. **获取最新数据**（使用 Exa `web_search_exa` 工具）  
   对每个阶段的关键指标搜索最新值：
   - 阶段 1：JOLTS 辞职率、招聘率（BLS 最近月度）
   - 阶段 2：底层 50% 超额储蓄（SF Fed）
   - 阶段 3：信用卡余额增速（G.19）、BNPL 食品占比（AFRM 最近季报）
   - 阶段 4：WMT/DG/MCD/CMG 最近季度同店数据（SEC 10-Q）
   - 阶段 5：CC 逾期率（NY Fed 季度）、汽车贷逾期（SUBN）
   - 阶段 6：次级信贷准入率（Fed SCE）
   - 阶段 7：Manheim 二手车批发、Eviction Lab 驱逐
   - 阶段 8：ABI 破产申请（Chapter 7/11/13 拆分）
   - 阶段 9：JPM/BAC/SYF/BFH 最近 10-Q（NCO、ACL）
   - 阶段 10：非农、ISM PMI、LEI
   - 阶段 11：VIX、HY spread（FRED BAMLH0A0HYM2）、CAPE

3. **数据来源纪律**
   - 公司数据优先 **SEC EDGAR 10-K/10-Q/8-K 原文**（用 `🏛️` 图标标注）
   - 宏观数据优先 **FRED / BLS / NY Fed / SF Fed / BEA** 官方链接
   - 估算值、行业协会数据、无法核实的 → 标 `⚠️ 未核实`
   - 已核实的官方/财报数据 → 标 `✅ 已核实`

4. **产出 HTML 报告（V2 模版）**

   **直接运行生成脚本（推荐）：**
   ```bash
   python3 ~/.claude/skills/美股宏观分析框架/assets/generate_report.py $(date +%F)
   ```
   输出路径：不传目录参数时默认输出到**当前工作目录**，文件名 `YYYY-MM-DD-综合诊断.html`

   **脚本结构说明（`assets/generate_report.py`）：**
   - `STAGES` 列表（第107行起）：11个阶段，每阶段包含 `concl`（当前结论）、`logic`（判断思路）、`action`（行动建议）、`rows`（指标表格数据）
   - 每次更新时，只需修改 `STAGES` 中对应阶段的 `col_now`（当前值）、`concl`（结论文字）
   - 图表数据在 `CHART_JS` 块（`mk('c1a', ...)` 等），更新最新周期的最后几个数据点

   **V2 模版特性：**
   - 每阶段：三段式结论（📌结论 / 🧠思路 / ⚡行动）→ 图表（上方）→ 指标表格（下方）
   - **23张三周期对比图**（阶段5独立3图：COF / AXP / 汽车贷）
   - X轴 `Y+0…Y+10`，`Y+5 ★2000/2008` / `Y+6 当前2026` 黄色高亮大圆点
   - 每张图带参考线（触发线 / 警戒线 / 危机线）
   - 顶部 diff 表格（本周 vs 上周变化）、传导链全景条、触发器状态表

5. **数据更新方法（每次生成前）：**

   打开 `assets/generate_report.py`，定位到对应阶段（搜索 `"t1"` ~ `"t11"`），修改：
   - `concl` 字符串 → 本周最新结论
   - 表格 `rows` 中的 `col_now` → 最新数据值
   - `CHART_JS` 中该阶段的 `ds('2020–2026', [...当前值...])` → 补充最新数据点

6. **触发器检查**  
   对照 `references/决策规则.md` 的 T1-T8 升级触发器和 R1-R6 缓和触发器，标出哪些已触发、哪些接近。在报告顶部 diff 表格和触发器状态表中体现。

### 工作流 B：周五 09:03 自动诊断

触发后自动执行工作流 A，且额外：
- 第 0 步：**diff 上周五的报告**，如有上周报告，重点标出哪些指标发生了实质变化（>10% 或跨档位）
- 第 7 步：生成后，**用一句话总结发给用户**："📅 YYYY-MM-DD 周度宏观诊断已生成（`路径`），X 个指标恶化，Y 个改善，触发器 Z 个新亮。"

## 重要的"不做"清单

- ❌ 不要用"平均"储蓄率/工资/消费支出当作结论，必须用分层数据
- ❌ 不要只说"逾期率下降了"，要交叉核对次级专营商数据和准入率
- ❌ 不要把单点值当成趋势，对比列必须是多年序列
- ❌ 不要在 HTML 里编造数据，无来源的标 `⚠️ 未核实`
- ❌ 不要复制粘贴上周报告的数据，每次都要重新搜索确认
- ❌ 不要省略 `🏛️` 图标——SEC 财报来源必须标识
